### שלב 2: אימות שדות ישראליים

לפני שממלאים טופס, תעשו אימות לכל הנתונים בפורמט ישראלי:

**אימות תעודת זהות:**
מספר תעודת זהות ישראלי הוא 9 ספרות עם ספרת ביקורת (וריאנט של Luhn):
```python
def validate_tz(id_number: str) -> bool:
    """אימות מספר תעודת זהות ישראלית."""
    id_str = id_number.zfill(9)
    if len(id_str) != 9 or not id_str.isdigit():
        return False
    total = 0
    for i, digit in enumerate(id_str):
        val = int(digit) * (1 + (i % 2))
        if val > 9:
            val -= 9
        total += val
    return total % 10 == 0
```

**פורמטים של מספרי טלפון ישראליים:**
| פורמט | דוגמה | הערות |
|-------|-------|-------|
| נייד | 05X-XXXXXXX | קידומות: 050, 051, 052, 053, 054, 055, 058 |
| קווי | 0X-XXXXXXX | אזורים: 02 (ירושלים), 03 (ת"א), 04 (חיפה), 08 (דרום), 09 (שרון) |
| בינלאומי | +972-5X-XXXXXXX | תסירו 0 מוביל, תוסיפו +972 |

**פורמט כתובת ישראלית:**
```
{שם רחוב} {מספר בית}, {דירה} (אופציונלי)
{שם עיר}, {מיקוד (7 ספרות)}
```

### שלב 3: הגדרת אוטומציית דפדפן (טפסים אונליין)

התקנה והגדרה של Playwright לפורטלים ממשלתיים בעברית (RTL):

```bash
pip install playwright
playwright install chromium
```

**דפוסים מרכזיים לפורטלי gov.il:**
```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)
    context = browser.new_context(locale="he-IL")
    page = context.new_page()
    page.goto("https://www.gov.il/he/service/...")

    # gov.il משתמש בטפסי React, תחכו לטעינה דינמית
    page.wait_for_selector('[data-testid="form-container"]', timeout=15000)

    # מילוי שדות טקסט RTL
    page.fill('input[name="firstName"]', "ישראל")
    page.fill('input[name="idNumber"]', "123456782")

    # תאריכים בפורמט ישראלי DD/MM/YYYY
    page.fill('input[name="birthDate"]', "15/03/1990")
```

### שלב 4: מילוי טפסי PDF (טפסים להורדה)

לטפסי PDF ממשלתיים עם שדות למילוי:

```python
# pikepdf (מומלץ)
import pikepdf
pdf = pikepdf.open("tofes_101.pdf")

# PyPDF2
from PyPDF2 import PdfReader, PdfWriter
reader = PdfReader("tofes_101.pdf")
fields = reader.get_fields()
```

### שלב 5: טיפול בדפוסי טפסים ממשלתיים נפוצים

**דוח שנתי (רשות המסים):**
1. ניווט לאזור האישי ברשות המסים
2. שדות נדרשים: מקורות הכנסה, ניכויים, זיכויים
3. צירוף טפסים דיגיטליים (טופס 106, טופס 857)
4. פורמט תאריך: תמיד DD/MM/YYYY

**תביעות ביטוח לאומי:**
1. ניווט לאזור האישי ב-btl.gov.il
2. זיהוי סוג התביעה (דמי לידה, נכות, אבטלה)
3. העלאת מסמכים תומכים (אישורים רפואיים, אישורי עבודה)
4. מעקב אחר סטטוס התביעה דרך הלוח האישי

**הגשות לרשם החברות:**
1. ניווט ל-ica.justice.gov.il
2. נדרש: מספר חברה, פרטי מורשה חתימה
3. הגשת דוח שנתי לרשם
4. הודעות על שינוי דירקטורים או בעלי מניות

### שלב 6: שליחה ואימות

אחרי מילוי הטופס:
1. **תצלמו צילום מסך לפני שליחה** -- לצילום הטופס המלא לבדיקת המשתמש
2. **תאמתו את כל השדות הנדרשים** -- טפסים ממשלתיים דוחים הגשות חלקיות
3. **תשמרו את מספר האישור** -- תמיד מוצג אחרי שליחה מוצלחת
4. **תורידו PDF קבלה** כשזמין
5. **תציינו זמן טיפול צפוי** -- רוב השירותים הממשלתיים מציינים זמן תגובה צפוי

### שלב 7: התאוששות משגיאות

בעיות נפוצות עם פורטלים ממשלתיים:
- **פקיעת סשן**: סשנים של gov.il פגים אחרי בערך 20 דקות חוסר פעילות
- **CAPTCHA**: חלק מהטפסים דורשים פתרון CAPTCHA ידני, תעצרו ותבקשו מהמשתמש
- **שגיאות אישורים**: פורטלים ממשלתיים יכולים להשתמש באישורי CA ישראליים
- **שעות עומס**: פורטל רשות המסים איטי בעונת ההגשות (מרץ-מאי)

## משאבים מצורפים

### סקריפטים
- `scripts/fill_form.py` -- עוזר לאימות שדות של טפסים ישראליים (תעודת זהות, טלפון, כתובת) ומילוי מבני נתונים של טפסים ממשלתיים נפוצים. להרצה: `python scripts/fill_form.py --help`

### חומרי עזר
- `references/gov-portals.md` -- רשימה מקיפה של כתובות פורטלים ממשלתיים, סוגי טפסים ומוסכמות שמות שדות. תסתכלו עליה כשאתם מזהים את הפורטל או הטופס הנכון למשימה.

## מלכודות נפוצות
- טפסים ממשלתיים בישראל דורשים הזנת טקסט בעברית בשדות ספציפיים. סוכנים עלולים לייצר נתוני טופס באנגלית בלבד, והמערכות הממשלתיות ידחו את זה.
- מספר תעודת זהות ישראלי הוא 9 ספרות עם ספרת ביקורת מסוג Luhn. סוכנים עלולים לייצר מספרים אקראיים בני 9 ספרות שנכשלים באימות ספרת הביקורת.
- הרבה טפסים ממשלתיים דורשים תאריך לידה גם בלוח עברי וגם גרגוריאני. סוכנים בדרך כלל מספקים רק את התאריך הגרגוריאני.
- חתימות דיגיטליות על טפסים ממשלתיים בישראל משתמשות במערכת אימות הזהות של gov.il. סוכנים לא יכולים לחתום על טפסים תוכנתית בלי שהמשתמש עובר אימות דרך gov.il.

## פתרון בעיות

### שגיאה: "פקיעת סשן" ב-gov.il
סיבה: סשנים של פורטלים ממשלתיים פגים אחרי חוסר פעילות ממושך
פתרון: תיכנסו מחדש ותמשיכו מהשלב האחרון שנשמר. תשמרו התקדמות חלקית לעתים קרובות.

### שגיאה: "תעודת זהות לא תקינה"
סיבה: מספר הזהות נכשל באימות ספרת ביקורת
פתרון: תריצו `validate_tz()` לפני הגשה. תוודאו 9 ספרות עם אפסים מובילים אם צריך.

### שגיאה: "טקסט עברי מוצג שגוי ב-PDF"
סיבה: ספריית ה-PDF לא תומכת בטקסט RTL או בגופנים עבריים
פתרון: תשתמשו ב-pikepdf עם גופנים עבריים מוטמעים. תוודאו שתבנית ה-PDF כוללת משאבי גופנים עבריים.

### שגיאה: "שדה טופס לא נמצא"
סיבה: טפסי PDF ממשלתיים משנים שמות שדות בין גרסאות
פתרון: תציגו קודם את כל השדות עם `reader.get_fields()`, ואז תתאימו לפי בדיקת תוויות השדות.
