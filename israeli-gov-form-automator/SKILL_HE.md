### שלב 2: אימות שדות ישראליים

לפני שממלאים טופס, תעשו אימות לכל הנתונים בפורמט ישראלי:

**אימות תעודת זהות:**
מספר תעודת זהות ישראלי הוא 9 ספרות עם ספרת ביקורת (וריאנט של Luhn):
```python
def validate_tz(id_number: str) -> bool:
    """אימות מספר תעודת זהות ישראלית."""
    id_str = id_number.zfill(9)
    if len(id_str) != 9 or not id_str.isdigit():
        return False
    if id_str == "000000000":
        return False  # עובר Luhn אבל הוא placeholder, לא ת״ז אמיתית
    total = 0
    for i, digit in enumerate(id_str):
        val = int(digit) * (1 + (i % 2))
        if val > 9:
            val -= 9
        total += val
    return total % 10 == 0
```

**פורמטים של מספרי טלפון ישראליים:**
| פורמט | דוגמה | הערות |
|-------|-------|-------|
| נייד | 05X-XXXXXXX | קידומות: 050, 051, 052, 053, 054, 055, 056 (HOT Mobile / Cellcom MVNO), 057 (פלאפון), 058, 059 (MVNO נוספים) |
| קווי | 0X-XXXXXXX | אזורים: 02 (ירושלים), 03 (ת"א), 04 (חיפה), 08 (דרום), 09 (שרון) |
| בינלאומי | +972-5X-XXXXXXX | תסירו 0 מוביל, תוסיפו +972 |

**פורמט כתובת ישראלית:**
```
{שם רחוב} {מספר בית}, {דירה} (אופציונלי)
{שם עיר}, {מיקוד (7 ספרות)}
```

לאימות מיקוד מוסמך השתמשו בשירות איתור מיקוד של דואר ישראל: `https://www.israelpost.co.il/zip_data.nsf/SearchZip`. תמיד לוודא שהמיקוד תואם לעיר לפני הגשה, אי-התאמה היא סיבת דחייה שכיחה.

**שדות בעברית בלבד מול שדות שמתירים אנגלית:** רוב הטפסים הממשלתיים דורשים טקסט בעברית. חריגים:

| הקשר | קלט מותר |
|------|----------|
| טפסי דרכון / אזרחים זרים (שמות) | אנגלית (לפי הדרכון) |
| הצעות שם חברה ברשם החברות | עברית עיקרית, תעתיק לועזי אופציונלי |
| IBAN / הפניות לחשבון בנק | תווים לטיניים (IL + ספרות) |
| דוא"ל וכתובות URL | לטיני |

בכל ספק, התאימו לכתיב ולכתב של המסמך הרשמי של המשתמש (תעודת זהות, דרכון, תעודת התאגדות).

### שלב 3: הגדרת אוטומציית דפדפן (טפסים אונליין)

התקנה והגדרה של Playwright לפורטלים ממשלתיים בעברית (RTL):

```bash
pip install playwright
playwright install chromium
```

**דפוסים מרכזיים לפורטלי gov.il:**
```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=False)
    context = browser.new_context(locale="he-IL")
    page = context.new_page()
    page.goto("https://www.gov.il/he/")

    # חשוב: סלקטורים לא יציבים בין פורטלי gov.il.
    # אל תניחו [data-testid="form-container"]. תמיד תספרו את ה-DOM
    # (page.content() או DevTools) ותגלו את הסלקטור הספציפי לפורטל
    # לפני שמסתמכים עליו.
    page.wait_for_selector("form, [role='main']", timeout=15000)

    # מילוי שדות טקסט RTL
    page.fill('input[name="firstName"]', "ישראל")
    page.fill('input[name="idNumber"]', "123456782")

    # תאריכים בפורמט ישראלי DD/MM/YYYY
    page.fill('input[name="birthDate"]', "15/03/1990")
```

### שלב 4: מילוי טפסי PDF (טפסים להורדה)

לטפסי PDF ממשלתיים עם שדות למילוי:

```python
# pikepdf (מומלץ)
import pikepdf
pdf = pikepdf.open("tofes_101.pdf")

# PyPDF2
from PyPDF2 import PdfReader, PdfWriter
reader = PdfReader("tofes_101.pdf")
fields = reader.get_fields()  # חובה: לספור שדות לפני שמנחשים שמות
```

תמיד הריצו `reader.get_fields()` ראשון. שמות שדות משתנים בין גרסאות טופס ובין משרדים, אסור להניח שם משימוש קודם.

**שמות שדות נפוצים בטפסים ממשלתיים:**

### שלב 5: טיפול בדפוסי טפסים ממשלתיים נפוצים

**טופס 101 (תיאום מס לעובד):**
מאז 2022 הטופס מחויב דיגיטלי דרך מערכות שכר של מעסיק ותהליך אישור דיגיטלי של רשות המסים (טופס 101 מקוון). זרימת PDF קיימת אבל היא חלופה ישנה. תהליך דיגיטלי: `https://www.gov.il/he/service/itc-application-for-electronic-101`.

**טופס 1301 / דוח שנתי (דוח אישי שנתי):**
הגשה מקוונת חובה מאז 2019 לרוב המגישים. הגשה בנייר מותרת רק לפנסיונרים ומגישים מסוימים מתחת לסף. מקור: כל זכות, "הגשת דוח שנתי למס הכנסה".

1. ניווט לאזור האישי ברשות המסים
2. שדות נדרשים: מקורות הכנסה, ניכויים, זיכויים
3. צירוף טפסים דיגיטליים (טופס 106, טופס 857)
4. פורמט תאריך: תמיד DD/MM/YYYY

**תביעות ביטוח לאומי:**
1. ניווט לאזור האישי ב-btl.gov.il
2. זיהוי סוג התביעה (דמי לידה, נכות, אבטלה)
3. העלאת מסמכים תומכים (אישורים רפואיים, אישורי עבודה)
4. מעקב אחר סטטוס התביעה דרך הלוח האישי

**הגשות לרשם החברות:**
1. ניווט ל-https://ica.justice.gov.il
2. נדרש: מספר חברה, פרטי מורשה חתימה
3. הגשת דוח שנתי לרשם
4. הודעות על שינוי דירקטורים או בעלי מניות

### שלב 7: חתימות דיגיטליות (נוף 2026)

שלוש שיטות מקובלות בפורטלים הממשלתיים:

1. **כרטיס חכם** מספקים מאושרים (Comsign, PersonalID). נדרש לזרימות מתקדמות במסים ולחלק מההגשות לרשם החברות.
2. **תעודת זהות ביומטרית + סיסמה**, אימות ראשי ל-my.gov.il ולאזור האישי.
3. **חתימה דיגיטלית מאומתת** לפי תקן הרשות הלאומית לדיגיטל.

סוכנים לא יכולים לחתום תוכנתית, על המשתמש לבצע את שלב החתימה אינטראקטיבית. מקורות: `https://www.govextra.gov.il/national-digital-agency/signature/home/`, `https://www.gov.il/he/departments/policies/digital_signature`.

### שלב 8: שליחה ואימות

אחרי מילוי הטופס:
1. **תצלמו צילום מסך לפני שליחה**, לצילום הטופס המלא לבדיקת המשתמש
2. **תאמתו את כל השדות הנדרשים**, טפסים ממשלתיים דוחים הגשות חלקיות
3. **תשמרו את מספר האישור**, תמיד מוצג אחרי שליחה מוצלחת
4. **תורידו PDF קבלה** כשזמין
5. **תציינו זמן טיפול צפוי**, רוב השירותים הממשלתיים מציינים זמן תגובה צפוי

### שלב 9: התאוששות משגיאות

בעיות נפוצות עם פורטלים ממשלתיים:
- **פקיעת סשן**: סשנים של gov.il פגים אחרי בערך 20 דקות חוסר פעילות
- **CAPTCHA**: חלק מהטפסים דורשים פתרון CAPTCHA ידני, תעצרו ותבקשו מהמשתמש
- **שגיאות אישורים**: פורטלים ממשלתיים יכולים להשתמש באישורי CA ישראליים
- **שעות עומס**: פורטל רשות המסים איטי בעונת ההגשות (מרץ-מאי)

## דוגמאות

### דוגמה 1: מילוי טופס 101
המשתמש אומר: "אני צריך למלא טופס 101 לעובד חדש"
פעולות:
1. תבדקו אם המעסיק עובד עם מערכת שכר שתומכת בטופס 101 דיגיטלי (מועדף מ-2022). חזרה ל-PDF רק אם לא.
2. אימות תעודת זהות של העובד
3. מילוי פרטים אישיים, מדרגת מס, נקודות זיכוי
4. יצירת PDF מלא לחתימת המעסיק (חלופה ישנה בלבד)
תוצאה: טופס 101 מוכן לאישור דיגיטלי או הגשת PDF

### דוגמה 2: הגשת תביעת דמי לידה
המשתמש אומר: "תעזרו לי להגיש תביעת דמי לידה בביטוח לאומי"
פעולות:
1. ניווט למדור דמי לידה ב-btl.gov.il
2. אימות זכאות: תקופת עבודה, נתוני שכר
3. מילוי טופס תביעה עם פרטים אישיים ופרטי מעסיק, אימות IBAN של חשבון הזיכוי
4. העלאת מסמכים נדרשים (אישור עבודה, שחרור מבית חולים)
תוצאה: תביעה שהוגשה עם מספר מעקב

### דוגמה 3: רישום חברה חדשה
המשתמש אומר: "אני רוצה לרשום חברה בע"מ חדשה"
פעולות:
1. ניווט לפורטל רשם החברות (https://ica.justice.gov.il)
2. מילוי הצעות שם (צריך 3 אפשרויות)
3. הזנת פרטי מייסדים, הון מניות, תקנון
4. חישוב אגרות רישום
תוצאה: אישור הגשה עם לוח זמנים צפוי לרישום

### דוגמה 4: עדכון כתובת במשרד הפנים
המשתמש אומר: "עברתי דירה ואני צריך לעדכן כתובת"
פעולות:
1. ניווט לשירות שינוי כתובת ב-gov.il
2. אימות פורמט כתובת חדשה (רחוב, עיר, מיקוד דרך דואר ישראל)
3. מילוי טופס שינוי כתובת עם הכתובת הישנה והחדשה
4. הגשה עם הזיהוי הנדרש
תוצאה: בקשת שינוי כתובת הוגשה עם מספר אישור

## משאבים מצורפים

### סקריפטים
- `scripts/fill_form.py`, עוזר לאימות שדות של טפסים ישראליים (תעודת זהות, טלפון, כתובת, IBAN) ומילוי מבני נתונים של טפסים ממשלתיים נפוצים. להרצה: `python scripts/fill_form.py --help`

### חומרי עזר
- `references/gov-portals.md`, רשימה מקיפה של כתובות פורטלים ממשלתיים, סוגי טפסים ומוסכמות שמות שדות. תסתכלו עליה כשאתם מזהים את הפורטל או הטופס הנכון למשימה.

## מלכודות נפוצות
- טפסים ממשלתיים בישראל דורשים הזנת טקסט בעברית בשדות ספציפיים. סוכנים עלולים לייצר נתוני טופס באנגלית בלבד, והמערכות הממשלתיות ידחו את זה.
- מספר תעודת זהות ישראלי הוא 9 ספרות עם ספרת ביקורת מסוג Luhn. סוכנים עלולים לייצר מספרים אקראיים בני 9 ספרות שנכשלים באימות ספרת הביקורת, וגם `000000000` עובר את Luhn אבל הוא placeholder, לא ת״ז אמיתית.
- הרבה טפסים ממשלתיים דורשים תאריך לידה גם בלוח עברי וגם גרגוריאני. סוכנים בדרך כלל מספקים רק את התאריך הגרגוריאני.
- חתימות דיגיטליות על טפסים ממשלתיים בישראל משתמשות במערכת אימות הזהות של gov.il. סוכנים לא יכולים לחתום על טפסים תוכנתית בלי שהמשתמש עובר אימות דרך gov.il.
- סלקטורים בפורטלי gov.il לא יציבים בין שירותים. אסור לעשות שימוש חוזר בסלקטור מפורטל אחר, תמיד לספור את ה-DOM קודם.

## פתרון בעיות

### שגיאה: "פקיעת סשן" ב-gov.il
סיבה: סשנים של פורטלים ממשלתיים פגים אחרי חוסר פעילות ממושך
פתרון: תיכנסו מחדש ותמשיכו מהשלב האחרון שנשמר. תשמרו התקדמות חלקית לעתים קרובות.

### שגיאה: "תעודת זהות לא תקינה"
סיבה: מספר הזהות נכשל באימות ספרת ביקורת, או שזה placeholder כמו `000000000`
פתרון: תריצו `validate_tz()` לפני הגשה. תוודאו 9 ספרות עם אפסים מובילים אם צריך.

### שגיאה: "טקסט עברי מוצג שגוי ב-PDF"
סיבה: ספריית ה-PDF לא תומכת בטקסט RTL או בגופנים עבריים
פתרון: תשתמשו ב-pikepdf עם גופנים עבריים מוטמעים. תוודאו שתבנית ה-PDF כוללת משאבי גופנים עבריים.

### שגיאה: "שדה טופס לא נמצא"
סיבה: טפסי PDF ממשלתיים משנים שמות שדות בין גרסאות
פתרון: תציגו קודם את כל השדות עם `reader.get_fields()`, ואז תתאימו לפי בדיקת תוויות השדות.
